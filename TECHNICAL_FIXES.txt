BLACKBOXAPP BUILD FIXES - TECHNICAL REFERENCE
==============================================

PROBLEM SUMMARY:
----------------
The BlackBoxApp uses Cordova 10.1.2 with older gradle configurations that are incompatible with modern Android SDK build-tools (31+). The main issue: Cordova required the legacy DX bytecode compiler tool which was removed from Android Build Tools.

FIXES APPLIED:
--------------

1. CORDOVA GRADLE VERSION DETECTION (cordova.gradle)
   ─────────────────────────────────────────────────
   Problem:
   - Original code limited build-tools search to versions within the same major version
   - If minimum required = 30.0.3, it would only look for versions 30.x.x
   - Modern SDK only has 36.x and 37.x available
   - Result: "No installed build tools found" error

   Solution:
   - Modified version range calculation
   - Capped maximum version at 37.0.0 to avoid incompatible versions
   - Search algorithm now: minVersion ≤ found ≤ 37.0.0
   
   Files Modified:
   - platforms/android/CordovaLib/cordova.gradle (Line ~60)

   Detailed Change:
   OLD:  def maxVersion = new Version((minBuildToolsVersion.getMajor() + 1) + ".0.0")
   NEW:  def maxVersion = new Version("37.0.0")
   
   OLD:  .findAll { it.isHigherThan('0.0.0') && it.isLowerThan(maxVersion) }
   NEW:  .findAll { it.isHigherThan('0.0.0') && !it.isLowerThan(minBuildToolsVersionString) && it.isLowerThan(maxVersion) }


2. D8 COMPILER SETUP (Instead of DX)
   ──────────────────────────────────
   Problem:
   - DX tool was removed in Android Build Tools 31+
   - Newer build-tools (36.x, 37.x) only have D8
   - Cordova gradle script looks for dx.jar and dx.bat

   Solution:
   - Create symbolic link from dx.jar → d8.jar
   - Enable D8 compiler in gradle.properties
   - Add dexOptions to disable pre-dex optimization
   
   Implementation:
   Command (run as Administrator):
   cd C:\Users\[You]\AppData\Local\Android\Sdk\build-tools\36.1.0\lib
   mklink dx.jar d8.jar
   
   Files Modified:
   - Created symbolic link: dx.jar → d8.jar
   - Enabled in: platforms/android/gradle.properties
   - Added dexOptions in: platforms/android/app/build.gradle


3. GRADLE CONFIGURATION (gradle.properties)
   ────────────────────────────────────────
   Problem:
   - Gradle wasn't configured to use D8 for bytecode compilation
   - Pre-dex optimization conflicted with D8

   Solution:
   Added properties:
   - android.enableD8=true          (Enable D8 compiler)
   - usePreDex false                (Disable pre-dex)
   
   File: platforms/android/gradle.properties
   Added lines:
   android.enableD8=true


4. DEX OPTIONS CONFIGURATION (app/build.gradle)
   ───────────────────────────────────────────
   Problem:
   - Default dex settings conflicted with modern build-tools
   - Pre-dex optimization caused build failures

   Solution:
   Added dexOptions block to android config:
   
   dexOptions {
       usePreDex false
   }
   
   File: platforms/android/app/build.gradle
   Location: Inside android { } block, after compileOptions


COMPLETE FIX SEQUENCE:
---------------------

If manually applying fixes (instead of using batch script):

Step 1: Verify Build-Tools Installation
  Command: dir %ANDROID_SDK_ROOT%\build-tools
  Expected: Directory with 36.1.0 or similar

Step 2: Apply cordova.gradle Patches
  File: platforms/android/CordovaLib/cordova.gradle
  Changes: See section 1 above
  Backup: cordova.gradle.bak

Step 3: Create DX⟷D8 Symlink
  Command (Admin): 
    cd %ANDROID_SDK_ROOT%\build-tools\36.1.0\lib
    mklink dx.jar d8.jar
  OR (If symlink fails):
    copy d8.jar dx.jar

Step 4: Update gradle.properties  
  File: platforms/android/gradle.properties
  Add: android.enableD8=true

Step 5: Update app/build.gradle
  File: platforms/android/app/build.gradle
  Add dexOptions in android block (see section 4)

Step 6: Clean Build
  Command: rmdir /s /q platforms\android\build

Step 7: Build
  Command: npm install && npm run android

Common Error Messages & Fixes:

Error: "No installed build tools found"
  → Need to patch cordova.gradle (see section 1)

Error: "Build-tool X is missing DX"
  → Need to create dx.jar symlink (see section 2)

Error: "No signature of method android()"
  → Need to ensure buildscript block is before android block

Error: "deprecated NDK warning"
  → This is safe to ignore, gradle will self-correct

Error: "FAILURE: Build failed... Unexpected element"
  → Android SDK XML parsing issue, usually resolves on retry


GRADLE PROPERTIES REFERENCE:
---------------------------

Set in: platforms/android/gradle.properties

org.gradle.jvmargs=-Xmx2048m         # JVM memory
android.useAndroidX=true              # AndroidX support
android.enableJetifier=true           # Jetpack compatibility
android.enableD8=true                 # D8 bytecode compiler
android.useDeprecatedNdk=false        # NDK handling (avoid setting)

AUTOMATION SCRIPT DETAILS:
--------------------------

The batch script (setup-and-build-android.bat) automates all procedures:

Phase 1: Verification
  - Checks Node.js installation
  - Locates Android SDK
  - Verifies environment variables

Phase 2: Installation
  - npm install (fetches dependencies)

Phase 3: Patching (cordova.gradle)
  - Backs up original
  - Applies version detection fix
  - Preserves other functionality

Phase 4: D8 Setup
  - Detects build-tools versions
  - Creates dx.jar symlink
  - Falls back to copy if symlink fails

Phase 5: Configuration
  - Updates gradle.properties
  - Modifies app/build.gradle
  - Preserves existing settings

Phase 6: Building
  - Cleans old build artifacts
  - Executes npm run android
  - Deploys to connected device
